## What is an API Gateway?

An **API Gateway** is a single entry point for all client requests. It handles cross-cutting concerns like authentication, rate limiting, and request routing.

---

## Without vs With API Gateway

```mermaid
flowchart LR
    subgraph "Without Gateway"
        C1["Client"] --> SA1["Service A"]
        C1 --> SB1["Service B"]
        C1 --> SC1["Service C"]
    end
    subgraph "With Gateway"
        C2["Client"] --> GW["API Gateway"]
        GW --> SA2["Service A"]
        GW --> SB2["Service B"]
        GW --> SC2["Service C"]
    end
```

---

## Key Functions

| **Function** | **Description** |
|-------------|-----------------|
| Routing | Route requests to appropriate services |
| Authentication | Verify identity (JWT, OAuth) |
| Authorization | Check permissions |
| Rate Limiting | Prevent abuse |
| Load Balancing | Distribute traffic |
| Caching | Cache responses |
| Request Transformation | Modify requests/responses |
| SSL Termination | Handle HTTPS |
| Monitoring | Log and trace requests |

---

## Request Flow

```mermaid
flowchart TD
    C["Client Request"] --> GW["API Gateway"]
    GW --> SSL["1. SSL Termination"]
    SSL --> Auth["2. Authentication"]
    Auth --> RL["3. Rate Limiting"]
    RL --> Route["4. Request Routing"]
    Route --> LB["5. Load Balancing"]
    LB --> Cache["6. Response Caching"]
    Cache --> B["Backend Services"]
```

## Patterns

### Request Routing

```mermaid
flowchart LR
    GW["Gateway"] -->|"/api/users/*"| US["User Service"]
    GW -->|"/api/orders/*"| OS["Order Service"]
    GW -->|"/api/products/*"| PS["Product Service"]
```

### Request Aggregation

Combine multiple service calls into one response:

```mermaid
flowchart TD
    C["Client: GET /api/dashboard"] --> GW["Gateway"]
    GW --> P["GET /users/profile"]
    GW --> O["GET /orders/recent"]
    GW --> N["GET /notifications/unread"]
    P & O & N --> R["Combined Response"]
```

### Protocol Translation

```mermaid
flowchart LR
    C["Client"] -->|"REST"| GW["Gateway"] -->|"gRPC"| S["Services"]
```

## Rate Limiting Strategies

| **Strategy** | **Description** |
|-------------|-----------------|
| Token Bucket | Tokens refill over time |
| Leaky Bucket | Constant rate output |
| Fixed Window | Count per time window |
| Sliding Window | Rolling time window |

---

## Considerations

### Pros

- Single entry point
- Centralized cross-cutting concerns
- Protocol translation
- Response aggregation

### Cons

- Single point of failure
- Added latency
- Additional complexity
- Can become bottleneck

---

## Interview Tips

- Explain the main functions (routing, auth, rate limiting)
- Discuss aggregation pattern for BFF (Backend for Frontend)
- Know popular solutions: Kong, AWS API Gateway
- Mention as potential bottleneck/SPOF
- Compare with service mesh
